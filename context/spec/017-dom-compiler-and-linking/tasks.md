## Notes for the Implementation Agent

- **Module-DSL `.directive` is OUT of scope.** All registration in this spec goes through `$compileProvider.directive` from a config block. The future `module.directive(...)` shorthand is a separate roadmap bullet.
- **Built-in directives are OUT of scope.** No `ng-if`, `ng-class`, `ng-bind`, `ng-click`, etc. Tests register custom directives inline.
- **No transclusion, no templates, no controllers.** Directives that declare `transclude`, `template`, `templateUrl`, `controller`, `controllerAs`, `bindToController`, `require`, `replace`, `multiElement` are silently accepted at registration time (forward-compat) but produce no observable behavior. Document this in `src/compiler/README.md`.
- **No string input to `$compile`.** Callers parse strings to DOM nodes themselves (e.g., via `new DOMParser()` or a `<template>` element).
- **`$rootScope` is NOT registered on `ngModule` here.** Tests construct `Scope.create()` directly. The `$rootScope` factory registration is part of the deferred "Application Bootstrap" roadmap item.
