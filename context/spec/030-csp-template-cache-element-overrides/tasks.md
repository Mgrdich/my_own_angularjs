## Notes for the Implementation Agent

- **No new `EXCEPTION_HANDLER_CAUSES` entry.** Both new error classes (`NgRefBadExpressionError`, `NgRefNoControllerError`) route via the existing `'$compile'` cause token. The tuple stays at 10.
- **No compiler-walker changes.** E-restrict matching already derives candidate names from `element.tagName.toLowerCase()` (`directive-collector.ts:112`); `'script'` and `'a'` pass registration validation as-is.
- **The `expression-assign.ts` extraction is a pure move** — the isolate-binding suite gates it with zero test edits. `ngRef` inherits the `=` binding's assignability semantics (Identifier | MemberExpression, `ensurePath` auto-create) by construction.
- **The anchor guard never digests** — `preventDefault` only, no scope mutation, so the spec-026 `$apply` try/catch workaround is NOT needed.
- **None of the five directives is structural** (`transclude: 'element'`) — the spec-027 terminal-cutoff gap is not implicated; `<a ng-if="…">` compositions are safe.
- **Tick task checkboxes in the SAME commit as the implementation** per `CLAUDE.md`.
