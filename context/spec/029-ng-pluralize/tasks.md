## Notes for the Implementation Agent

- **No new `EXCEPTION_HANDLER_CAUSES` entry.** Both new error classes (`NgPluralizeNoRuleDefinedError`, `NgPluralizeBadOffsetError`) route via the existing `'$compile'` cause token. The tuple stays at 10.
- **`pluralCat` is a required-field `.d.ts` break** — accepted deliberately (pre-1.0, upstream parity; an optional field would silently fall back to English rules). The two custom-locale test literals are the only in-repo call sites needing updates.
- **No `$log` service exists in this project.** Upstream's `$log.debug` for missing rules is replaced by handler-routed `NgPluralizeNoRuleDefinedError` — a documented divergence, reported once per key *transition*, never on NaN counts.
- **The `(countExpr) - offset` rewrite is parenthesized** — a documented micro-deviation from upstream's bare concatenation; identical semantics for all upstream-legal inputs.
- **The `when` map is static** (one link-time `$eval`, upstream parity); the count and embedded `{{expr}}` bindings are live.
- **`ng-pluralize` is a leaf text-writer** — not `transclude: 'element'`, not terminal; it composes with structural hosts like `ng-class` does and does not widen the spec-027 same-element gap.
- **Tick task checkboxes in the SAME commit as the implementation** per `CLAUDE.md` — the co-modification of `tasks.md` + `src/` in one commit is the traceability signal the audit looks for.
