## Notes for the Implementation Agent

- **No new `EXCEPTION_HANDLER_CAUSES` entry.** The `'$compile'` token (already in the 10-tuple from spec 018) covers every new error site — `ng-include` fetch failures, `ng-switch` missing-parent throws, `ng-controller` unknown-name throws, and the reused `MultipleTranscludeDirectivesError`. The `onload` modifier's evaluation throws naturally route via `'watchListener'` (also already in the tuple). The tuple stays at 10.
- **`transclude: 'element'` is the load-bearing prerequisite.** Slices 3, 5, and 6 all depend on Slice 2. Slice 1 (`ng-init`) and Slice 4 (`ng-controller`) do not — they could ship without it. The slice ordering puts `ng-init` first (smallest atomic value, no foundation needed), then the foundation, then the directives that need it.
- **`runControllerSeam` widening is a central change** (Slice 4). The existing two branches (`bindToController` deferred-alias + eager `$controller(directive.controller, …)`) must continue to behave identically. The third branch only fires when `directive.controller === { __attributeSource: 'ngController' }`. Type-union the field to keep the sentinel from leaking to other consumers.
- **Comment-placeholder cleanup is a directive-author responsibility.** Each of the three structural directives (`ng-if`, the `ng-switch` children, `ng-include`) registers `addElementCleanup(placeholder, () => destroyElementScope(currentClonedRoot))` on every successful `$transclude(...)` call. Without this, a parent `destroyElementScope` reaching the placeholder cannot tear the active clone down (Comment nodes have no `children` HTMLCollection).
- **Stale-fetch sentinel in `ng-include` is load-bearing.** Without it, a fetch resolving AFTER the surrounding scope was destroyed installs a subtree into a torn-down DOM hole — memory leak via dangling scope references. The `currentLoadToken !== thisToken` check on the resolve path is the gate.
- **`ng-switch` parent-controller pattern.** Children register via `require: '^ngSwitch'`; the parent's `$watch` listener orchestrates clone install/teardown. Children install no clones themselves. This is AngularJS-canonical.
