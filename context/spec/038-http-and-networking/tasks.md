## Notes for the Implementation Agent

- **Verification is vitest + the five gates** — `$http` is DOM-free except the JSONP `<script>` transport (covered by jsdom); no browser MCP / external services needed. Stub `$httpBackend` (or `fetch`) for the non-JSONP request tests; use vitest fake timers for timeout/cancellation.
- **`$http` returns `$q` promises → digest integration is FREE.** Resolving the deferred schedules a digest via `$q`'s `$rootScope.$evalAsync` seam — do NOT call `$apply` in `$http`.
- **No `EXCEPTION_HANDLER_CAUSES` change** — `$http` failures are promise REJECTIONS (FS §2.7), never routed through `$exceptionHandler`. The tuple stays **13**; do not add a token.
- **Keep the factories pure** — `createHttp` / `createHttpBackend` / `createCacheFactory` take injected seams (`$q`, `$httpBackend`, `$cacheFactory`, `$$cookieReader`, the fetch/timer globals) and are unit-testable without an injector; the `ngModule` registrations bind the real services.
- **Reuse, don't reinvent:** `isSameOrigin` (from `@sce`) for XSRF; the `$templateRequest` `inFlight: Map` dedup pattern; the `$templateCache` Map shape for `$cacheFactory`; the `$FilterProvider` resolve-at-`$get` pattern (`'$injector'` injectable) for interceptors; the `ng-include` lazy `$injector.has('$sce')` probe for JSONP gating.
- **Deferred (documented, NOT this spec):** `$httpProvider.useApplyAsync`, `$cacheFactory` LRU/`capacity`, upload/download progress (fetch limitation).
- **Tick task checkboxes in the SAME commit as the implementation** per `CLAUDE.md`.
