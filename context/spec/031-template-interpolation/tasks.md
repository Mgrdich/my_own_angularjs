## Notes for the Implementation Agent

- **No new `EXCEPTION_HANDLER_CAUSES` entry.** Per-expression throws route through `$interpolate`'s handler (`'$interpolate'` / `'$filter'`); `$set` observer-notification throws route as `'$compile'`. Tuple stays at 10.
- **No new DI dependency on `$compile`.** `interpolate` is already a wired collaborator (`compile.ts:400`); SCE callbacks are already wired into `$interpolate` by `$InterpolateProvider.$get`, so trusted-context routing needs only the 3rd arg.
- **Single watch per interpolated attribute.** The shared `$$interpolators` cache is the source of truth â€” the eager pass installs, `$observe` reuses. Never install twice.
- **`ng-*` attribute rewrite is benign.** Eager interpolation also classifies `ng-href="{{url}}"` and writes the resolved value back into the *`ng-href`* attribute; `ng-href`/`ng-src` still own the real `href`/`src` (different normalized name). If a spec-025 assertion pins the literal `{{}}` on the `ng-*` attribute, update it â€” writing the resolved value is benign and closer to AngularJS.
- **Transclusion text must clone correctly.** The cloneMap parallel-walk widening (Slice 1) is load-bearing â€” without it a transcluded text node's watch attaches to the never-inserted master node (clone shows literal `{{}}` + a leaked watch). Tests pin per-clone bindings.
- **First-render flash is accepted** (parity); `ng-cloak`/`ng-bind`/`ng-href`/`ng-src` remain the flash-free options.
