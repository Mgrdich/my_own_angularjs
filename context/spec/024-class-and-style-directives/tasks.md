## Notes for the Implementation Agent

- **No new `EXCEPTION_HANDLER_CAUSES` entry.** The `'watchListener'` token covers every Slice 1–4 error site (every directive's diff cycle runs inside a `$watchCollection` listener; throws are reported via the existing path). The tuple stays at 10.
- **`flattenClassExpression` is shared, but module-private.** Used by all three `ng-class*` directives in `ng-class.ts`; not exported from `@compiler/index` or the root barrel.
- **`installClassWatcher` is the engine.** `ngClassDirective` calls it without a gate; `ngClassEvenDirective` / `ngClassOddDirective` call it with `(scope) => !!scope.$even` / `!!scope.$odd`. The helper installs the `$watchCollection` on the expression AND (when a gate is present) a separate `scope.$watch('$even' | '$odd', refire)` to handle the gate flipping without the expression changing.
- **`ng-style` writes via `setProperty` / `removeProperty`.** NOT via `element.style.cssText = …` — preserves consumer-shipped inline styles by design.
- **All four directives register on `ngModule` only.** Same DI-registration-only precedent as spec 018's `ngTransclude` and spec 023's seven directives. No exports from `@compiler/index`.
- **Tick task checkboxes in the SAME commit as the implementation** per `CLAUDE.md` — the co-modification of `tasks.md` + `src/` in one commit is the traceability signal the audit looks for.
