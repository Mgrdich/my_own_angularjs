## Notes for the Implementation Agent

- **No new `EXCEPTION_HANDLER_CAUSES` entry.** The four new error classes (`NgRepeatBadIteratorExpressionError`, `NgRepeatBadIdentifierError`, `NgRepeatBadAliasError`, `NgRepeatDuplicateKeyError`) all route via the existing `'$compile'` cause token. The tuple stays at 10.
- **`transclude: 'element'` is the load-bearing prerequisite — already shipped in spec 027 Slice 2.** No changes to the foundation are needed; `ng-repeat` is a straightforward consumer.
- **The reconciliation engine grows across slices 3 → 4 → 5 → 6.** Each slice adds ONE capability (default identity → track by + reuse → object iteration → alias publication). The duplicate-detection contract and the per-row scope shape are stable from Slice 3.
- **WeakMap-based identity tracker is the deliberate divergence from AngularJS.** AngularJS injects a non-enumerable `$$hashKey` onto user objects; we don't. Tests that AngularJS upstream wrote against `$$hashKey` introspection must be translated to identity-based assertions.
- **`$animate` integration is deferred to Phase 4** — synchronous DOM mutations only. Matches the spec 027 precedent for `ng-show` / `ng-hide` / `ng-class`.
- **Same-element conflict with other structural directives** (e.g. `<li ng-repeat="…" ng-if="…">`) is the documented spec 027 gap — the spec-017 same-element terminal cutoff drops the lower-priority directive instead of producing `MultipleTranscludeDirectivesError`. Slice 7's integration test pins the actually-observable behavior; resolving the gap is out of scope here.
- **Tick task checkboxes in the SAME commit as the implementation** per `CLAUDE.md` — the co-modification of `tasks.md` + `src/` in one commit is the traceability signal the audit looks for.
