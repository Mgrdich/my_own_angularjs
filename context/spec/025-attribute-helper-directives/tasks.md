## Notes for the Implementation Agent

- **No new `EXCEPTION_HANDLER_CAUSES` entry.** The `'watchListener'` and `'$evalAsync'` tokens (inherited from `scope.$watch` and the static-attribute `$observe` first-fire path respectively) cover every Slice 1–3 error site. The tuple stays at 10.
- **Both factory helpers are module-private.** `createUrlAliasDirective` and `createBooleanAliasDirective` live inside `ng-attribute-aliases.ts` and are NOT exported from `@compiler/index`. Only the 8 generated factories are exported from the file itself, and none of those are exported from `@compiler/index` either (DI-registration-only — same precedent as `ngTransclude`, all spec 023 / 024 directives).
- **The 8 directives all live in ONE file.** `ng-attribute-aliases.ts` mirrors AngularJS-1.x's `ngAttributeAliasDirectives` pattern — one file, two `forEach`-style generators, 8 outputs.
- **`attrs.$set` does the right thing.** Truthy → `setAttribute`, falsy → `removeAttribute`. The directives just call `$set` with the resolved value; the framework handles the DOM write + observer notification.
- **Priority 99 (URL) and 100 (boolean) match AngularJS-canonical.** Don't change these to make tests pass — they're load-bearing for AngularJS-1.x parity if a consumer combines these with other prioritized directives.
- **Tick task checkboxes in the SAME commit as the implementation** per `CLAUDE.md` — the co-modification of `tasks.md` + `src/` in one commit is the traceability signal the audit looks for.
