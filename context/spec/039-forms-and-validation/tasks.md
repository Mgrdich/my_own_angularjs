## Notes for the Implementation Agent

- **Verification is Vitest + jsdom + the five gates** — forms are DOM directives but jsdom covers them (the existing `$compile` test pattern); no browser MCP or external services required. Async-validator and `ngModelOptions` timing use vitest fake timers.
- **Keep `NgModelController`/`FormController` parity-faithful** — port behavior and tests from `angular/angular.js/test/ng/directive/` (`ngModelSpec.js`, `inputSpec.js`, `formSpec.js`, `selectSpec.js`, `ngOptionsSpec.js`).
- **No new `EXCEPTION_HANDLER_CAUSES` token** — directive-side errors reuse `'$compile'`; tuple stays at 13. Grep for hardcoded `=== 13` before any change.
- **Single `input` directive + internal `inputType` registry** (parity) — not one directive per type. `textarea` delegates to the `text` handler.
- **Slice 3/6 timezone seam** — date handlers ship with a default/local timezone in Slice 3; the `ngModelOptions.timezone` wiring generalizes it in Slice 6. Keep date parse/format isolated in `input-date.ts` to make that handoff clean.
- **Model write-back via `buildParentWriter`** (`@compiler/expression-assign`) — don't reimplement assignment; non-assignable `ng-model` routes `'$compile'`.
- **Async validators use `$q`** — `$$runValidators` runs sync `$validators` first, async only if all sync pass; a per-run generation counter cancels stale async passes; deregister watches/listeners/timers on `$destroy`.
- **Synchronous state-class toggles** — `$animate` integration is deferred to Phase 4; toggle via `classList`, append-only, never stripping author classes (the `ng-class` guarantee).
- **Tick task checkboxes in the SAME commit as the implementation** per `CLAUDE.md`.
